# Home Guardian 移动端 UI 功能点文档

本文档用于交给另一个 AI 生成移动端 UI 图，目标是覆盖当前移动端已实现功能，并补充后端已具备、建议在 UI 方案中预留的模块。

## 1. 产品定位

- 产品名称：Home Guardian
- 产品性质：智能家居移动控制台 / 家庭环境监控与设备控制 PWA
- 目标用户：家庭管理员、普通家庭成员、访客型用户
- 核心价值：实时看家里状态、看历史数据、处理告警、控制设备、配置自动化和通知

## 3. 设计输出要求

给另一个 AI 生成 UI 图时，建议按以下原则输出：

- 以手机竖屏为主，默认 iPhone 13/14 尺寸即可
- 需要覆盖登录流、主导航流、详情流、表单流、设置流
- 每个页面至少给出正常态；核心页面同时给出空状态、加载态、异常态或操作反馈态
- 需要体现实时感：在线状态、告警数量、图表趋势、最近触发时间
- 需要体现家庭空间感：房间、设备分组、家庭场景、设备位置
- 需要体现管理感：启停开关、批量操作、筛选、确认弹窗、危险操作

## 5. 全局交互与状态系统

### 5.1 登录与会话

- 登录后进入首页
- 请求自动携带 Bearer Token
- Token 过期后自动使用 refresh_token 刷新 access_token
- 刷新失败后自动清空登录态并跳回登录页
- 退出登录需要确认弹窗

### 5.2 实时消息能力

WebSocket 需要在 UI 图中体现以下消息带来的变化：

- device_status：设备在线/离线状态即时变化
- telemetry：设备最新指标值即时刷新
- alert：新告警产生，未读数量增加
- notification：系统通知弹出 Toast，同时未读数量增加

### 5.3 全局状态反馈

所有主要页面都建议体现以下状态：

- Loading：骨架屏 / 卡片骨架 / 图表骨架
- Empty：暂无设备、暂无告警、暂无规则、暂无命令记录、暂无通知渠道、暂无数据
- Error：网络错误、提交失败、操作失败
- Success：登录成功、已保存、已删除、已发送、已确认、已解决
- Confirm：删除确认、批量确认、批量解决、退出登录确认

### 5.4 权限与角色感知

后端是 RBAC + 位置作用域模型，移动端 UI 图建议预留：

- 无权限时隐藏危险操作或置灰
- 仅展示当前用户可见设备 / 房间 / 告警
- 对管理类模块保留“普通成员可看、管理员可配”的层级感

1. 页面头部
2. 告警摘要卡

- 是否启用

- 设备

- 动作类型
  - 发送指令 send_command
  - 发送通知 send_notification

#### 当动作为“发送指令”

- 目标设备
- 指令名 action

### 提交动作

- 创建自动化
- 保存修改

### UI 建议

- 这是一个条件流编辑页，建议做成分段卡片，而不是普通长表单
- 可用“触发条件 -> 执行动作”的视觉流程线来表现自动化逻辑
